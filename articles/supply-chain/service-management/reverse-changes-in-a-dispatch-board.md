# <a name="reverse-changes-in-a-dispatch-board"></a><span data-ttu-id="d3e3f-103">Izmaiņu atsaukšana nosūtīšanas pārvaldē</span><span class="sxs-lookup"><span data-stu-id="d3e3f-103">Reverse changes in a dispatch board</span></span> 

[!include [banner](../includes/banner.md)]


<span data-ttu-id="d3e3f-104">Šajā tēmā aprakstīts, kā atsaukt nesaglabātās nosūtīšanas pārvaldē veiktās izmaiņas.</span><span class="sxs-lookup"><span data-stu-id="d3e3f-104">This topic describes how to reverse unsaved modifications that you make in a dispatch board.</span></span> <span data-ttu-id="d3e3f-105">Piemēram, pakalpojuma aktivitātei tiek piešķirts darbinieks, saglabāt informācija, bet vēlāk šai pakalpojuma aktivitātei tiek nolemts piešķirt citu darbinieku.</span><span class="sxs-lookup"><span data-stu-id="d3e3f-105">For example, you assign a worker to a service activity, save the information, and then later decide to assign a different worker to the service activity.</span></span> <span data-ttu-id="d3e3f-106">Nosūtīšanas pārvaldē tiek mainīts darbinieks, un pirms izmaiņu saglabāšanas tiek iegūta informācija, ka tikko piešķirtais darbinieks nav pieejams.</span><span class="sxs-lookup"><span data-stu-id="d3e3f-106">You modify the worker in the dispatch board, and then, before saving the change, learn that the worker just assigned is not available.</span></span> <span data-ttu-id="d3e3f-107">Nesaglabātās izmaiņas var atsaukt, lai pakalpojuma pasūtījumam tiktu atkārtoti piešķirts sākotnējais darbinieks.</span><span class="sxs-lookup"><span data-stu-id="d3e3f-107">You can reverse the unsaved modification so that the original worker is reassigned to the service order.</span></span>

<span data-ttu-id="d3e3f-108">Lai nosūtīšanas pārvaldē atsauktu nesaglabātās izmaiņas, veiciet tālāk norādītās darbības.</span><span class="sxs-lookup"><span data-stu-id="d3e3f-108">Use the following steps to reverse unsaved changes in a dispatch board:</span></span>

1.  <span data-ttu-id="d3e3f-109">Noklikšķiniet uz **Pakalpojumu pārvaldība** \> **Periodiskās darbības** \> **Dispečera pults**.</span><span class="sxs-lookup"><span data-stu-id="d3e3f-109">Click **Service management** \> **Periodic** \> **Dispatch board**.</span></span>

2.  <span data-ttu-id="d3e3f-110">Veidlapas **Dispečera pults** laukos ievadiet atbilstošo informāciju un noklikšķiniet uz **Labi**.</span><span class="sxs-lookup"><span data-stu-id="d3e3f-110">In the **Dispatch board** form, enter relevant information in the fields, and then click **OK**.</span></span> 

3.  <span data-ttu-id="d3e3f-111">Lai atsauktu pēdējās nesaglabātās izmaiņas, noklikšķiniet uz **Atsaukt**.</span><span class="sxs-lookup"><span data-stu-id="d3e3f-111">To reverse the most recent change that is not saved, click **Undo**.</span></span>

4.  <span data-ttu-id="d3e3f-112">Lai atgrieztu vairākas izmaiņas, kas nav saglabātas, turpiniet klikšķināt uz **Atsaukt**, līdz katra no atceļamajām izmaiņām ir tikusi atgriezta.</span><span class="sxs-lookup"><span data-stu-id="d3e3f-112">To reverse a series of changes that are not saved, continue clicking **Undo** until each change that you want to discard is reversed.</span></span>
