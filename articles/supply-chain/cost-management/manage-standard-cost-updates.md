# <a name="manage-standard-cost-updates"></a><span data-ttu-id="f1939-103">Standarta izmaksu atjauninājumu pārvaldība</span><span class="sxs-lookup"><span data-stu-id="f1939-103">Manage standard cost updates</span></span>

[!include [banner](../includes/banner.md)]

<span data-ttu-id="f1939-104">Standarta izmaksu datu atjauninājumus var pārvaldīt, izmantojot divas dažādas pieejas — vienas versijas pieeju vai divu versiju pieeju.</span><span class="sxs-lookup"><span data-stu-id="f1939-104">Updates to standard cost data can be managed by using two different approaches - the one-version approach or the two-version approach.</span></span>

<span data-ttu-id="f1939-105">Vienas versijas pieejā tiek izmantota viena izmaksu aprēķināšanas versija, kas satur visus izmaksu ierakstus.</span><span class="sxs-lookup"><span data-stu-id="f1939-105">The one-version approach uses a single costing version that contains all cost records.</span></span> <span data-ttu-id="f1939-106">Šie ieraksti ietver sākotnējās izmaksas un visus izmaksu atjauninājumus.</span><span class="sxs-lookup"><span data-stu-id="f1939-106">These records include the original costs and all cost updates.</span></span>

<span data-ttu-id="f1939-107">Divu versiju pieejā tiek izmantota viena versija, kas satur sākotnējo izmaksu ierakstus, un otra versija, kas satur visu izmaksu atjauninājumu ierakstus.</span><span class="sxs-lookup"><span data-stu-id="f1939-107">The two-version approach uses one version that contains records of the original costs and a second version that contains records of all cost updates.</span></span> <span data-ttu-id="f1939-108">Primārā divu versiju pieejas priekšrocība ir skaidrais izmaksu atjauninājumu un izsekošanas attēlojums atsevišķā izmaksu novērtēšanas versijā, neietekmējot sākotnējo izmaksu novērtēšanas versiju.</span><span class="sxs-lookup"><span data-stu-id="f1939-108">A primary advantage of the two-version approach is the clear delineation and tracking of cost updates in a separate costing version, without affecting the original costing version.</span></span> <span data-ttu-id="f1939-109">Divu versiju pieeju var izmantot, lai identificētu vairākus inkrementālus atjauninājumus, kuram katram ir atsevišķa izmaksu aprēķināšanas versija, kas satur inkrementālo izmaksu ierakstus.</span><span class="sxs-lookup"><span data-stu-id="f1939-109">The two-version approach can be used to identify multiple incremental updates, where each incremental update has a separate costing version that contains the incremental cost records.</span></span>

## <a name="example"></a><span data-ttu-id="f1939-110">Paraugs</span><span class="sxs-lookup"><span data-stu-id="f1939-110">Example</span></span>

<span data-ttu-id="f1939-111">Tālāk sniegtajā piemērā ir parādīts, kā vienas versijas un divu versiju pieeju var izmantot standarta izmaksu atjaunināšanai ražošanas vidē.</span><span class="sxs-lookup"><span data-stu-id="f1939-111">The following example illustrates how the one-version and two-version approaches can be used for updating standard costs in a manufacturing environment.</span></span> <span data-ttu-id="f1939-112">Piemēram, atjauninājumi, kas attiecas uz jauniem krājumiem vai kļūdu labojumiem.</span><span class="sxs-lookup"><span data-stu-id="f1939-112">For example, updates that reflect new items or error corrections.</span></span> <span data-ttu-id="f1939-113">Pieņemsim, ka viena izmaksu aprēķināšanas versija attiecas uz standarta izmaksām pašreizējā gadā.</span><span class="sxs-lookup"><span data-stu-id="f1939-113">Assume that a single costing version represents the standard costs for the current year.</span></span> <span data-ttu-id="f1939-114">Šī versija identifikators ir 2020-STD.</span><span class="sxs-lookup"><span data-stu-id="f1939-114">The identifier for this version is 2020-STD.</span></span> <span data-ttu-id="f1939-115">Versija 2020-STD satur visu krājumu pašreizējās aktīvās izmaksas.</span><span class="sxs-lookup"><span data-stu-id="f1939-115">Version 2020-STD contains the current active costs for all items.</span></span> <span data-ttu-id="f1939-116">Turklāt tā satur visas ar maršrutēšanu saistītās izmaksu kategorijas un pieskaitāmo izmaksu aprēķināšanas formulas, kas bija zināmas 2020. gada sākumā.</span><span class="sxs-lookup"><span data-stu-id="f1939-116">Additionally, it contains all routing-related cost categories and overhead calculation formulas that were known at the start of the year 2020.</span></span> <span data-ttu-id="f1939-117">2020-STD ir sākotnējā izmaksu aprēķināšanas versija.</span><span class="sxs-lookup"><span data-stu-id="f1939-117">2020-STD is the original costing version.</span></span>

- <span data-ttu-id="f1939-118">**Vienas versijas pieeja izmaksu datu atjauninājumiem** — vienas versijas pieejā sākotnējā izmaksu aprēķināšanas versija 2020-STD satur visus izmaksu ierakstus.</span><span class="sxs-lookup"><span data-stu-id="f1939-118">**One-version approach to cost data updates** − In the one-version approach, the original costing version 2020-STD contains all cost records.</span></span> <span data-ttu-id="f1939-119">Izmaksu atjauninājumi tiek reģistrēti versijā 2020-STD, un tiem tiek iestatīts statuss Gaida.</span><span class="sxs-lookup"><span data-stu-id="f1939-119">Cost updates are recorded in 2020-STD and set to a status of Pending.</span></span> <span data-ttu-id="f1939-120">Lai atainotu labojumus, gaidītās izmaksas var manuāli ievadīt, ja izmaksas ir saistītas ar jauniem iegādātajiem krājumiem, vai arī aprēķināt, ja izmaksas ir saistītas ar saražotu krājumu.</span><span class="sxs-lookup"><span data-stu-id="f1939-120">The pending costs can be manually entered for new purchased items, or they can be calculated for a manufactured item to reflect corrections.</span></span> <span data-ttu-id="f1939-121">Ja tiek izmantota vienas versijas pieeja, MK aprēķiniem nav nepieciešams atkāpšanās datu avots, jo visas aktīvās izmaksas ir ietvertas izmaksu noteikšanas versijā.</span><span class="sxs-lookup"><span data-stu-id="f1939-121">When the one-version approach is used, the BOM calculations do not require a fallback data source because all active costs are contained in the costing version.</span></span> <span data-ttu-id="f1939-122">Kad gaidāmās izmaksas kļūst aktīvas, sākotnēja izmaksu aprēķināšanas versija 2020-STD atkal satur visas pašreizējās aktīvās izmaksas.</span><span class="sxs-lookup"><span data-stu-id="f1939-122">After the pending costs become active, the original costing version 2020-STD will again contain all the current active costs.</span></span>
- <span data-ttu-id="f1939-123">**Divu versiju pieeja izmaksu datu atjauninājumiem** — divu versiju pieejai nepieciešama papildu izmaksu aprēķināšanas versija, kas satur tikai izmaksu for this version is 2020-STD-CHANGES.</span></span> <span data-ttu-id="f1939-125">Izmaksu atjauninājumi tiek reģistrēti versijā 2020-STD-CHANGES, un tiem tiek iestatīts statuss Gaida.</span><span class="sxs-lookup"><span data-stu-id="f1939-125">Cost updates are recorded in 2020-STD-CHANGES and are set to a status of Pending.</span></span> <span data-ttu-id="f1939-126">Ja tiek izmantota divu versiju pieeja, saražoto krājumu gaidāmo izmaksu MK aprēķiniem ir nepieciešams atkāpšanās datu avots.</span><span class="sxs-lookup"><span data-stu-id="f1939-126">With the two-version approach, the BOM calculations of pending costs for manufactured items require a fallback data source.</span></span> <span data-ttu-id="f1939-127">Tas ir tāpēc, ka papildu izmaksu aprēķināšanas versija 2020-STD-IZMAIŅAS satur tikai izmaksu datu apakškopu.</span><span class="sxs-lookup"><span data-stu-id="f1939-127">This is because the additional costing version 2020-STD-CHANGES contains only a subset of cost data.</span></span> <span data-ttu-id="f1939-128">Atkāpšanās var būt izteikta kā aktīvās izmaksas vai kā izmaksu aprēķināšanas versija 2020-STD, jo tās abas norāda versijā 2020-STD-IZMAIŅAS neietverto izmaksu datu avotu.</span><span class="sxs-lookup"><span data-stu-id="f1939-128">The fallback can be expressed as the active costs or as the costing version 2020-STD, 2020-STD will not be affected.</span></span> <span data-ttu-id="f1939-130">Ja tiek izmantota divu versiju pieeja, ir jāiestata sākotnējās izmaksu aprēķināšanas versijas bloķēšanas politikas, lai <span data-ttu-id="f1939-133">Šajā piemērā tika izmantota viena papildu izmaksu aprēķināšanas versija, lai pārvaldītu atjauninājumus par 2020. gadu.</span><span class="sxs-lookup"><span data-stu-id="f1939-133">This example used one additional costing version for managing updates throughout the year 2020.</span></span> <span data-ttu-id="f1939-134">Var izmantot vairākas izmaksu aprēķināšanas versijas, piemēram, atsevišķu versiju katram atjauninājumu paketei.</span><span class="sxs-lookup"><span data-stu-id="f1939-134">More than one additional costing version can be used, such as a separate version for each batch of updates.</span></span> <span data-ttu-id="f1939-135">Ja tiek izmantotas vairākas papildu izmaksu aprēķināšanas versijas, atkāpšanās ir jāizsaka kā aktīvās izmaksas, jo aktīvās izmaksas ir ietvertas vairākās izmaksu aprēķināšanas versijās.</span><span class="sxs-lookup"><span data-stu-id="f1939-135">When more than one additional costing is used, the fallback must be expressed as the active costs, because the active costs are spread over multiple costing versions.</span></span>
