# <a name="replenishment-overview"></a><span data-ttu-id="84db4-103">Papildināšanas pārskats</span><span class="sxs-lookup"><span data-stu-id="84db4-103">Replenishment overview</span></span>

[!include [banner](../includes/banner.md)]

<span data-ttu-id="84db4-104">Šajā tēmā ir aprakstītas papildināšanas stratēģijas, kas ir pieejamas noliktavām, kuras izmanto noliktavas vadībā pieejamo funkcionalitāti.</span><span class="sxs-lookup"><span data-stu-id="84db4-104">This topic describes the replenishment strategies that are available for warehouses that use the functionality that is available in Warehouse management.</span></span> <span data-ttu-id="84db4-105">Informācija šajā tēmā neattiecas uz noliktavu risinājumu, kas ir pieejams modulī Krājumu vadība.</span><span class="sxs-lookup"><span data-stu-id="84db4-105">The information in this topic doesn't apply to the warehousing solution that is available in Inventory management.</span></span>

<span data-ttu-id="84db4-106">Ir pieejamas tālāk minētās papildināšanas stratēģijas.</span><span class="sxs-lookup"><span data-stu-id="84db4-106">The following replenishment strategies are available:</span></span>

- <span data-ttu-id="84db4-107">**Kopuma pieprasījuma papildināšana** — šī stratēģija izveido papildināšanas darbu izejošajiem pasūtījumiem vai kravām, ja krājums nav pieejams, kad kopums izveido darbu.</span><span class="sxs-lookup"><span data-stu-id="84db4-107">**Wave demand replenishment** – This strategy creates replenishment work for outbound orders or loads if inventory isn't available when the wave creates work.</span></span> <span data-ttu-id="84db4-108">Piemēram, papildināšanas darbu var izveidot, ja kopuma apstrādāšanas laikā pārdošanas pasūtījumā pieprasītais daudzums nav pieejams.</span><span class="sxs-lookup"><span data-stu-id="84db4-108">For example, replenishment work can be created if the quantity that is required for a sales order isn't available when a wave is processed.</span></span>
- <span data-ttu-id="84db4-109">**Minimālā/maksimālā papildināšana** — šī stratēģija izmanto minimālos un maksimālos dimensiju ierobežojumus, lai noteiktu novietojumu papildināšanas laiku.</span><span class="sxs-lookup"><span data-stu-id="84db4-109">**Min/Max replenishment** – This strategy uses minimum and maximum stocking limits to determine when locations should be replenished.</span></span> <span data-ttu-id="84db4-110">Krājumu un novietojumu kritērijus nosaka krājumi, kas tiek novērtēti attiecībā uz papildināšanu.</span><span class="sxs-lookup"><span data-stu-id="84db4-110">The item and location criteria define the inventory that is evaluated for replenishment.</span></span> <span data-ttu-id="84db4-111">Minimālās/maksimālās papildināšanas veidnes ir galvenais mehānisms optimālu līmeņu uzturēšanai izdošanas vietās.</span><span class="sxs-lookup"><span data-stu-id="84db4-111">Min/Max replenishment templates are the primary mechanism for maintaining optimal levels in picking locations.</span></span> <span data-ttu-id="84db4-112">Lai palīdzētu garantēt, ka ir kopuma pieprasījumam atbilstošs daudzums izdošanai pieejamo krājumu, papildus minimālās/maksimālās papildināšanas cikliem varat izmantot pieprasījuma papildināšanu.</span><span class="sxs-lookup"><span data-stu-id="84db4-112">To help guarantee that enough pick face inventory is available to meet wave demand, you can use demand replenishment as a supplement between Min/Max replenishment cycles.</span></span>
- <span data-ttu-id="84db4-113">**Kravas pieprasījuma papildināšana** — šī stratēģija summē pieprasījumu pēc vairākām kravām un izveido papildināšanas darbu, kas ir nepieciešams, lai būtu pietiekami daudz krājumu attiecīgajās izdošanas vietās.</span><span class="sxs-lookup"><span data-stu-id="84db4-113">**Load demand replenishment** – This strategy sums the demand for several loads and creates the replenishment work that is required in order to stock the relevant picking locations.</span></span> <span data-ttu-id="84db4-114">Šī stratēģija palīdz garantēt, ka izveidotās kravas var izdot noliktavā pēc to izlaišanas.</span><span class="sxs-lookup"><span data-stu-id="84db4-114">This strategy helps guarantee that the loads that are created can be picked in the warehouse after they are released.</span></span>
- <span data-ttu-id="84db4-115">**Tūlītēja papildināšana** — šī stratēģija krājumus papildina, pirms tiek izpildīts kopums, ja notiek nesekmīgs sadalījums kādai novietojuma direktīvas rindai, kurai ir papildinājuma veidne.</span><span class="sxs-lookup"><span data-stu-id="84db4-115">**Immediate replenishment** – This strategy replenishes inventory before a wave is run if allocation fails for a location directive line that has a replenishment template.</span></span> 

<span data-ttu-id="84db4-116">Visas četras stratēģijas izveido papildināšanas darbu, pamatojoties uz kādu papildināšanas veidni.</span><span class="sxs-lookup"><span data-stu-id="84db4-116">All four strategies create replenishment work, based on a replenishment template.</span></span>

## <a name="wave-demand-replenishment"></a><span data-ttu-id="84db4-117">Kopuma pieprasījuma papildināšana</span><span class="sxs-lookup"><span data-stu-id="84db4-117">Wave demand replenishment</span></span>

<span data-ttu-id="84db4-120">Novietojuma direktīvās tiek izmantotas, lai noteiktu, kuri novietojumi ir jāpapildina.</span><span class="sxs-lookup"><span data-stu-id="84db4-120">Location directives are used to determine which location should be replenished.</span></span> <span data-ttu-id="84db4-121">Šīs novietojuma direktīvas jūs saistāt ar papildināšanas veidni, izmantojot lauku **Direktīvas kods**.</span><span class="sxs-lookup"><span data-stu-id="84db4-121">You link these location directives to the replenishment template by using the **Directive code** field.</span></span> <span data-ttu-id="84db4-122">Ja lauks **Direktīvas kods** nav iestatīts, tiek izmantoti vaicājumi, lai noteiktu, kura novietojuma direktīva ir jāizmanto.</span><span class="sxs-lookup"><span data-stu-id="84db4-122">If the **Directive code** field isn't set, queries are used to determine which location directive should be used.</span></span> <span data-ttu-id="84db4-123">Ņemiet vērā — ja direktīvas kods nav norādīts papildināšanas veidnē, bet novietojuma direktīvai ir direktīvas kods, tad novietojuma direktīva tiks ignorēta, pat ja vaicājums uz šo noliktavas direktīvu ir pareizs.</span><span class="sxs-lookup"><span data-stu-id="84db4-123">Note that if a directive code isn't specified in the replenishment template, and the location directive has a directive code, the location directive will be ignored, even if the query on the location directive is correct.</span></span> <span data-ttu-id="84db4-124">Izdošanas novietojuma direktīvas tiek izmantotas, lai noteiktu, kur iegūt krājumus papildināšanai.</span><span class="sxs-lookup"><span data-stu-id="84db4-124">Pick location directives are used to determine where to get inventory for the replenishment.</span></span>

<span data-ttu-id="84db4-133">Pieprasījuma papildināšana tiek atbalstīta pārdošanas pasūtījumiem, pārsūtīšanas pasūtījumiem, ražošanas pasūtījumiem un Kanban.</span><span class="sxs-lookup"><span data-stu-id="84db4-133">Demand replenishment is supported for sales orders, transfer orders, production orders, and kanbans.</span></span> 

## <a name="minmax-replenishment"></a><span data-ttu-id="84db4-134">Minimālā/maksimālā papildināšana</span><span class="sxs-lookup"><span data-stu-id="84db4-134">Min/Max replenishment</span></span>
