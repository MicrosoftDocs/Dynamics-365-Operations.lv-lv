<span data-ttu-id="9b56f-104">Šajā rakstā aprakstīts, kā izmantot darba veidnes un novietojuma direktīvas, lai noteiktu, kā un kur darbs tiek veikts noliktavā.</span><span class="sxs-lookup"><span data-stu-id="9b56f-104">This article describes how to use work templates and location directives to determine how and where work is carried out in the warehouse.</span></span>

<span data-ttu-id="9b56f-105">Norādījumi, ko noliktavas darbinieki saņem mobilajā ierīcē, ir atkarīgi no darba veidnēm, kuras iestatāt programmatūrā Microsoft Dynamics 365 for Finance and Operations, lai definētu dažādos noliktavas procesus un uzdevumus.</span><span class="sxs-lookup"><span data-stu-id="9b56f-105">The instructions that warehouse workers receive on a mobile device are determined by the work templates that you set up in Microsoft Dynamics 365 for Finance and Operations to define the various warehouse processes and tasks.</span></span> <span data-ttu-id="9b56f-106">Darba veidnes nosaka, kā darbs tiek veikts katram noliktavas procesam.</span><span class="sxs-lookup"><span data-stu-id="9b56f-106">Work templates determine how the work is performed for each warehouse process.</span></span> <span data-ttu-id="9b56f-107">Saistot novietojuma direktīvu ar darba veidnēm, var palīdzēt garantēt, ka darbs notiek noteiktās noliktavu fiziskās daļās.</span><span class="sxs-lookup"><span data-stu-id="9b56f-107">By linking a location directive to work templates, you can help guarantee that work occurs in specific physical areas of the warehouses.</span></span>

<span data-ttu-id="9b56f-111">Darba veidnes sastāv no virsraksta un saistītām rindām.</span><span class="sxs-lookup"><span data-stu-id="9b56f-111">Work templates consists of a header and associated lines.</span></span> <span data-ttu-id="9b56f-112">Katra darba veidne ir paredzēta īpašam *darba pasūtījuma veidam*.</span><span class="sxs-lookup"><span data-stu-id="9b56f-112">Each work template is for a specific *work order type*.</span></span> <span data-ttu-id="9b56f-113">Daudzi darba pasūtījuma tipi ir saistīti ar avota dokumentiem, piemēram, pirkšanas vai pārdošanas pasūtījumiem.</span><span class="sxs-lookup"><span data-stu-id="9b56f-113">Many work order types are associated with source documents, such as purchase or sales orders.</span></span> <span data-ttu-id="9b56f-114">Tomēr, citi darba pasūtījumu tipi veido atsevišķus noliktavas procesus, piemēram, cikla inventarizāciju.</span><span class="sxs-lookup"><span data-stu-id="9b56f-114">However, other work order types represent separate warehouse processes, such as cycle counting.</span></span> <span data-ttu-id="9b56f-115">*Darbu kopu ID* ļauj organizēt darbu grupās.</span><span class="sxs-lookup"><span data-stu-id="9b56f-115">The *work pool ID* lets you organize work into groups.</span></span> 

<span data-ttu-id="9b56f-116">Iestatījumus darba virsraksta definīcijā var izmantot, lai noteiktu, kad jāizveido jauns darba gabals.</span><span class="sxs-lookup"><span data-stu-id="9b56f-116">The settings in the work header definition can be used to determine when a new piece of work should be created.</span></span> <span data-ttu-id="9b56f-117">Piemēram, var iestatīt maksimālo izdošanas rindu skaitu un maksimālo plānoto izdošanas laiku.</span><span class="sxs-lookup"><span data-stu-id="9b56f-117">For example, you can set a maximum number of pick lines and a maximum expected pick time.</span></span> <span data-ttu-id="9b56f-118">Pēc tam, ja darbs pārdošanas pasūtījuma izdošanas procesam pārsniedz kādu no šīm vērtībām, šis darbs tiek sadalīts divos darba gabalos.</span><span class="sxs-lookup"><span data-stu-id="9b56f-118">Then, if the work for a sales order picking process exceeds either of those values, that work is split into two pieces of work.</span></span> 

<span data-ttu-id="9b56f-119">Darba rindas attēlo fiziskos uzdevumus, kas ir nepieciešami, lai varētu apstrādāt darbu.</span><span class="sxs-lookup"><span data-stu-id="9b56f-119">The work lines represent the physical tasks that are required in order to process the work.</span></span> <span data-ttu-id="9b56f-120">Piemēram, izejošajam noliktavas procesam var būt darba rinda krājumu paņemšanai noliktavā un cita rinda šo krājumu novietošanai sagatavošanas zonā.</span><span class="sxs-lookup"><span data-stu-id="9b56f-120">For example, for an outbound warehouse process, there might be a work line for picking up the items within the warehouse and another line for putting those items into a staging area.</span></span> <span data-ttu-id="9b56f-121">Pēc tam var būt papildu rinda krājumu izdošanai no sagatavošanas posmiem un cita rinda krājumu ievietošanai kravas automašīnā iekraušanas procesa ietvaros.</span><span class="sxs-lookup"><span data-stu-id="9b56f-121">There can then be an additional line for picking the items from staging, and another line for putting the items into a truck as part of the loading process.</span></span> <span data-ttu-id="9b56f-122">Varat iestatīt *direktīvas kodu* darba veidnes rindās.</span><span class="sxs-lookup"><span data-stu-id="9b56f-122">You can set a *directive code* on work template lines.</span></span> <span data-ttu-id="9b56f-123">Direktīvas kods ir saistīts ar novietojuma direktīvu un tāpēc palīdz nodrošināt, ka noliktavas darbs tiek apstrādāts pareizā novietojumā noliktavā.</span><span class="sxs-lookup"><span data-stu-id="9b56f-123">A directive code is linked to a location directive and therefore helps guarantee that the warehouse work is processed in the correct location in the warehouse.</span></span> 

<span data-ttu-id="9b56f-124">Vaicājumu var iestatīt, lai kontrolētu, kad tiek izmantota konkrētā darba veidne.</span><span class="sxs-lookup"><span data-stu-id="9b56f-124">You can set up a query to control when a particular work template is used.</span></span> <span data-ttu-id="9b56f-125">Piemēram, var iestatīt ierobežojumu, lai noteiktu veidni varētu izmantot darbam tikai noteiktā noliktavā.</span><span class="sxs-lookup"><span data-stu-id="9b56f-125">For example, you can set a limitation so that a particular template can be used for work only in a specific warehouse.</span></span> <span data-ttu-id="9b56f-126">Vai arī var būt vairākas veidnes, kas tiek izmantotas, lai izveidotu darbu izejošā pārdošanas pasūtījuma apstrādei atkarībā no pārdošanas izcelsmes.</span><span class="sxs-lookup"><span data-stu-id="9b56f-126">Alternatively, you might have several templates that are used to create work for outbound sales order processing, depending on the sales origin.</span></span> <span data-ttu-id="9b56f-127">Lai noteiktu secību, kādā tiek novērtētas pieejamās darbu veidnes, sistēma izmanto lauku **Kārtas numurs**.</span><span class="sxs-lookup"><span data-stu-id="9b56f-127">The system uses the **Sequence number** field to determine the order that the available work templates are assessed in.</span></span> <span data-ttu-id="9b56f-128">Tādēļ, ja jums ir ļoti konkrēts vaicājums konkrētai darba veidnei, tam ir jāpiešķir zems kārtas numurs.</span><span class="sxs-lookup"><span data-stu-id="9b56f-128">Therefore, if you have a very specific query for a particular work template, you should give it a low sequence number.</span></span> <span data-ttu-id="9b56f-129">Pēc tam šis vaicājums tiks novērtēts pirms citiem, vispārīgākiem vaicājumiem.</span><span class="sxs-lookup"><span data-stu-id="9b56f-129">That query will then be evaluated before other, more general queries.</span></span> 

<span data-ttu-id="9b56f-130">Lai apturētu vai pauzētu darba procesu, var izmantot iestatījumu **Pārtraukt darbu** darba rindā.</span><span class="sxs-lookup"><span data-stu-id="9b56f-130">To stop or pause a work process, you can use the **Stop work** setting on the work line.</span></span> <span data-ttu-id="9b56f-131">Tādā gadījumā, darbiniekam, kurš veic darbu, netiks lūgts veikt nākamās darba rindas darbību.</span><span class="sxs-lookup"><span data-stu-id="9b56f-131">In that case, the worker who is performing the work won't be asked to perform the next work line step.</span></span> <span data-ttu-id="9b56f-132">Lai pārietu uz nākamo darbību, šim darbiniekam vai citam darbiniekam jāatlasa darbs vēlreiz.</span><span class="sxs-lookup"><span data-stu-id="9b56f-132">To move on to the next step, that worker or another worker must select the work again.</span></span> <span data-ttu-id="9b56f-133">Varat arī atdalīt uzdevumus darba gabalā, izmantojot citu *darba klases ID* darba veidnes rindās.</span><span class="sxs-lookup"><span data-stu-id="9b56f-133">You can also separate the tasks within a piece of work by using a different *work class ID* on the work template lines.</span></span>

## <a name="location-directives"></a><span data-ttu-id="9b56f-134">Novietojuma direktīvas</span><span class="sxs-lookup"><span data-stu-id="9b56f-134">Location directives</span></span>
<span data-ttu-id="9b56f-135">Novietojuma direktīvas ir nosacījumi, kas palīdz identificēt izdošanas un izvietošanas novietojumus krājumu kustībai.</span><span class="sxs-lookup"><span data-stu-id="9b56f-135">Location directives are rules that help identify pick and put locations for inventory movement.</span></span> <span data-ttu-id="9b56f-136">Piemēram, pārdošanas pasūtījuma transakcijā novietojuma direktīva nosaka, kur krājumi tiks izdoti un kur izdotie krājumi tiks izvietoti.</span><span class="sxs-lookup"><span data-stu-id="9b56f-136">For example, in a sales order transaction, a location directive determines where the items will be picked, and where the picked items will be put.</span></span> <span data-ttu-id="9b56f-137">Novietojuma direktīvas sastāv no virsraksta un saistītām rindām un tās izveido lapā **Novietojuma direktīvas**.</span><span class="sxs-lookup"><span data-stu-id="9b56f-137">Location directives consist of a header and associated lines, and you create them on the **Location directives** page.</span></span> 

<span data-ttu-id="9b56f-138">Virsrakstā katrai novietojuma direktīvai jābūt saistītai ar *darba pasūtījuma tipu*, kas norāda krājumu transakcijas tipu, kas tiks izmantots direktīvai, piemēram, pārdošanas pasūtījumus, papildināšanu vai izejmateriālu izdošanu.</span><span class="sxs-lookup"><span data-stu-id="9b56f-138">On the header, each location directive must be associated with a *work order type* that specifies the type of inventory transaction that the directive will be used for, such as sales orders, replenishment, or raw material picking.</span></span> <span data-ttu-id="9b56f-139">*Darba tips* norāda, vai novietojuma direktīva tiks izmantota izdošanas vai nodošanas darbam vai citam noliktavas procesam, piemēram, inventarizācijai vai krājumu statusa izmaiņām.</span><span class="sxs-lookup"><span data-stu-id="9b56f-139">The *work type* specifies whether the location directive will be used for picking or putting work, or for some other warehouse process, such as counting or inventory status changes.</span></span> <span data-ttu-id="9b56f-140">Jānorāda arī *vietne* un *noliktava*.</span><span class="sxs-lookup"><span data-stu-id="9b56f-140">You must also specify a *site* and a *warehouse*.</span></span> <span data-ttu-id="9b56f-141">*Direktīvas kodu*, ko norādāt virsrakstā, var izmantot, lai saistītu novietojuma direktīvu ar vienu vai vairākām darba veidnēm.</span><span class="sxs-lookup"><span data-stu-id="9b56f-141">A *directive code* that you specify on the header can be used to link the location directive to one or more work templates.</span></span> 

<span data-ttu-id="9b56f-142">Attiecībā uz darba veidnēm, varat iestatīt vaicājumu, lai noteiktu, kad tiek izmantota konkrēta novietojuma direktīva.</span><span class="sxs-lookup"><span data-stu-id="9b56f-142">As for work templates, you can set up a query to determine when a particular location directive is used.</span></span> <span data-ttu-id="9b56f-143">Piemēram, varat norādīt ka tad, kad pārdošanas pasūtījuma izcelsme ir e-komercija, krājums ir jāizdod no atvēlētās zonas noliktavā.</span><span class="sxs-lookup"><span data-stu-id="9b56f-143">For example, you can specify that when e-commerce is the origin of a sales order, the inventory must be picked up from a dedicated area in the warehouse.</span></span> <span data-ttu-id="9b56f-144">Sistēma izmanto lauku **Secības numurs**, lai noteiktu secību, kādā tiek novērtētas pieejamas novietojuma direktīvas.</span><span class="sxs-lookup"><span data-stu-id="9b56f-144">The system uses the **Sequence number** field to determine the order that the available location directives are assessed in.</span></span> 

<span data-ttu-id="9b56f-145">Novietojuma direktīvas rindas iestata papildu ierobežojumus attiecībā uz novietojuma meklēšanas noteikumu lietošanu.</span><span class="sxs-lookup"><span data-stu-id="9b56f-145">The location directive lines set additional restrictions on the application of the location finding rules.</span></span> <span data-ttu-id="9b56f-146">Varat norādīt minimālo daudzumu un maksimālo daudzumu, kuram direktīva jāpiemēro, kā arī varat norādīt, ka direktīvai jābūt paredzētai noteiktai krājuma vienībai.</span><span class="sxs-lookup"><span data-stu-id="9b56f-146">You can specify a minimum quantity and a maximum quantity that the directive should apply to, and you can specify that the directive should be for a specific inventory unit.</span></span> <span data-ttu-id="9b56f-147">Piemēram, ja mērvienība ir paletes, krājumus paletēs var izvietot konkrētā novietojumā.</span><span class="sxs-lookup"><span data-stu-id="9b56f-147">For example, if the unit of measure is pallets, the items in pallets can be put in a specific location.</span></span> <span data-ttu-id="9b56f-148">Var arī norādīt, vai daudzumu var sadalīt starp vairākiem novietojumiem.</span><span class="sxs-lookup"><span data-stu-id="9b56f-148">You can also specify whether the quantity can be split across multiple locations.</span></span> <span data-ttu-id="9b56f-149">Tāpat kā novietojuma direktīvas virsraksts katrai novietojuma direktīvas rindai ir sērijas numurs, kas nosaka secību, kādā rindas tiek novērtētas.</span><span class="sxs-lookup"><span data-stu-id="9b56f-149">Like the location directive header, each location directive line has a sequence number that determines the order that the lines are assessed in.</span></span> 

<span data-ttu-id="9b56f-150">Novietojuma direktīvām ir viens papildu detalizācijas līmenis: *novietojuma direktīvas darbības*.</span><span class="sxs-lookup"><span data-stu-id="9b56f-150">Location directives have one additional level of detail: *location directive actions*.</span></span> <span data-ttu-id="9b56f-151">Var definēt vairākas novietojuma direktīvas darbības katrai rindai.</span><span class="sxs-lookup"><span data-stu-id="9b56f-151">You can define multiple location directive actions for each line.</span></span> <span data-ttu-id="9b56f-152">Un atkal — kārtas numurs tiek izmantots, lai noteiktu secību, kādā darbības tiek novērtētas.</span><span class="sxs-lookup"><span data-stu-id="9b56f-152">Once again, a sequence number is used to determine the order that the actions are assessed in.</span></span> <span data-ttu-id="9b56f-153">Šajā līmenī varat iestatīt vaicājumu, lai definētu, kā atrast vislabāko novietojumu noliktavā.</span><span class="sxs-lookup"><span data-stu-id="9b56f-153">On this level, you can set up a query to define how to find the best location in the warehouse.</span></span> <span data-ttu-id="9b56f-154">Varat arī izmantot iepriekš definētus **Stratēģijas** iestatījumus, lai atrastu optimālu novietojumu.</span><span class="sxs-lookup"><span data-stu-id="9b56f-154">You can also use predefined **Strategy** settings to find an optimal location.</span></span>

### <a name="example-of-the-use-of-location-directives"></a><span data-ttu-id="9b56f-155">Novietojuma direktīvas izmantošanas piemērs</span><span class="sxs-lookup"><span data-stu-id="9b56f-155">Example of the use of location directives</span></span>

<span data-ttu-id="9b56f-156">Šajā piemērā mēs apsvērsim pirkšanas pasūtījuma procesu, kur novietojuma direktīvai ir jāatrod brīva vieta noliktavā krājumiem, kas tikko reģistrēti saņemšanas dokā.</span><span class="sxs-lookup"><span data-stu-id="9b56f-156">For this example, we will consider a purchase order process where the location directive must find free capacity within a warehouse for inventory items that have just been registered at the receiving dock.</span></span> <span data-ttu-id="9b56f-157">Pirmkārt, mēs vēlamies mēģināt atrast brīvu vietu noliktavā, apvienojot rīcībā esošos krājumus.</span><span class="sxs-lookup"><span data-stu-id="9b56f-157">First, we want to try to find free capacity within the warehouse by consolidating with existing on-hand inventory.</span></span> <span data-ttu-id="9b56f-158">Ja konsolidācija nav iespējama, tad vēlamies atrast tukšu novietojumu.</span><span class="sxs-lookup"><span data-stu-id="9b56f-158">If consolidation isn't possible, we then want to find an empty location.</span></span> 

<span data-ttu-id="9b56f-159">Lai izmantotu šo scenāriju, mums jādefinē divas novietojuma direktīvas darbības.</span><span class="sxs-lookup"><span data-stu-id="9b56f-159">For this scenario, we must define two location directive actions.</span></span> <span data-ttu-id="9b56f-160">Pirmajai darbībai secībā ir jāizmanto stratēģija **Konsolidēt** un otrajai vajadzētu izmantot stratēģiju **Tukšs novietojums bez ienākoša darba**.</span><span class="sxs-lookup"><span data-stu-id="9b56f-160">The first action in the sequence must use the **Consolidate** strategy, and the second should use the **Empty location with no incoming work** strategy.</span></span> <span data-ttu-id="9b56f-161">Ja mēs noteiksim trešo darbību pārpildes scenārija apstrādei, ir iespējami divi rezultāti, kad noliktavā vairs nav vietas: darbu var izveidot pat tad, ja nav definēti novietojumi, vai darba izveides process var neizdoties.</span><span class="sxs-lookup"><span data-stu-id="9b56f-161">Unless we define a third action to handle an overflow scenario, two outcomes are possible when there is no more capacity in the warehouse: work can be created even though no locations are defined, or the work creation process can fail.</span></span> <span data-ttu-id="9b56f-162">Rezultātu nosaka iestatījumi lapā **Novietojuma direktīvas kļūmes**, kur varat izlemt, vai atlasīt opciju **Pārtraukt darbu, ja rodas ar novietojuma direktīvu saistīta kļūme** katram darba pasūtījuma tipam.</span><span class="sxs-lookup"><span data-stu-id="9b56f-162">The outcome is determined by the setup on the **Location directive failures** page, where you can decide whether to select the **Stop work on location directive failure** option for each work order type.</span></span>
