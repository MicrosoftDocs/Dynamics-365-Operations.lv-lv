<span data-ttu-id="1ded4-117">Notikumiem, kas pamatoti uz izmaiņām, var iestatīt pakešuzdevumu, kas izraisa notikuma apstrādi drīz pēc tam, kad sistēma ir pieteikusi notikumu.</span><span class="sxs-lookup"><span data-stu-id="1ded4-117">For change-based events, you can set up a batch job that triggers the processing of an event soon after the event is logged by the system.</span></span> <span data-ttu-id="1ded4-118">Ja pakešuzdevuma iestatījums nosaka tā biežāku atkārtošanu, lietotāji saņem brīdinājumus ātrāk, ja tiek veiktas izmaiņas.</span><span class="sxs-lookup"><span data-stu-id="1ded4-118">If you set up the batch job to recur more often, users receive their alerts sooner after changes occur.</span></span> <span data-ttu-id="1ded4-119">Tomēr bieža pakešveida apstrādes veikšana var negatīvi ietekmēt sistēmas veiktspēju.</span><span class="sxs-lookup"><span data-stu-id="1ded4-119">However, a high frequency for batch processing might adversely affect system performance.</span></span>

#### <a name="the-risks-of-low-batch-frequency"></a><span data-ttu-id="1ded4-127">Zema pakešuzdevumu biežuma risks</span><span class="sxs-lookup"><span data-stu-id="1ded4-127">The risks of low batch frequency</span></span>

<span data-ttu-id="1ded4-128">Ja tiek iestatīts ar notikumiem saistītā pakešveida apstrādes zems izpildes biežums, ar brīdinājuma kārtulu nosacījumiem saistītie dati var tikt izmainīti pirms pakešuzdevums ir apstrādāts.</span><span class="sxs-lookup"><span data-stu-id="1ded4-128">If you set up a low frequency for batch processing for change-based events, data that is relevant to the conditions in alert rules might be changed before the batch is processed.</span></span> <span data-ttu-id="1ded4-129">Tāpēc brīdinājumi var tikt zaudēti.</span><span class="sxs-lookup"><span data-stu-id="1ded4-129">Therefore, you might lose alerts.</span></span>
