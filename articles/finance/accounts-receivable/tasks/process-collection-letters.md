## <a name="control-collection-letters-at-the-customer-level"></a><span data-ttu-id="89600-145">Atgādinājuma vēstuļu pārvaldība debitora līmenī</span><span class="sxs-lookup"><span data-stu-id="89600-145">Control collection letters at the customer level</span></span>
<span data-ttu-id="89600-146">Ja transakciju līmenī ir iestatītas atgādinājuma vēstules, klientam var tikt izveidotas vairākas vēstules, kas balstītas uz transakciju vecumstruktūru.</span><span class="sxs-lookup"><span data-stu-id="89600-146">If collection letters are set up at the transaction level, multiple letters might be generated for a customer, based on transaction aging.</span></span> <span data-ttu-id="89600-147">Ja transkacijas parādās dažādās vēstules virknēs, katrai nokavēto traksakciju grupai tiks ģenerētas atsevišķas atgādinājuma vēstules klientam.</span><span class="sxs-lookup"><span data-stu-id="89600-147">If transactions appear in different letter sequences, separate collection letters will be generated for each group of overdue transactions for the customer.</span></span> <span data-ttu-id="89600-148">Tādēļ atsevišķs klients var saņemt, piemēram, vienu atgādinājuma vēstuli par transakcijām, kuru termiņš ir pārsniedzis 60 dienas, un citas atgādinājuma vēstules, kuru termiņš ir pārsniedzis 90 dienas.</span><span class="sxs-lookup"><span data-stu-id="89600-148">Therefore, an individual customer might receive, for example, one collection letter for transactions that are 60 days overdue and another collection letter for transactions that are 90 days overdue.</span></span>

<span data-ttu-id="89600-152">Atgādinājuma vēstules var iestatīt arī klientu līmenī.</span><span class="sxs-lookup"><span data-stu-id="89600-152">Collection letters can also be set up at the customer level.</span></span> <span data-ttu-id="89600-153">Šajā gadījumā tiks izsekots katras transakcijas atgādinājuma vēstules kods, bet atgādinājuma vēstuļu apstrādes pamatā būs viens atgādinājuma vēstuļu līmenis, kas saglabāts klientam.</span><span class="sxs-lookup"><span data-stu-id="89600-153">In this case, the collection letter code for each transaction is tracked, but collection letter processing will be based on a single collection letter level that is stored for the customer.</span></span> <span data-ttu-id="89600-154">Viena atgādinājuma vēstule ietvers visas klienta nokavētās transakcijas.</span><span class="sxs-lookup"><span data-stu-id="89600-154">The single collection letter will contain all the transactions that are overdue for the customer.</span></span> <span data-ttu-id="89600-155">Tā kā pagarinājuma dienas tagad tiek izsekotas klienta līmenī, nākamā atgādinājuma vēstule netiek nosūtīta, līdz nav pagājis nākamās atgādinājuma vēstules nosūtīšanai noteiktais pagarinājuma dienu skaits, pat ja transakcijas tiek nokavētas pēc pēdējās atgādinājuma vēstules nosūtīšanas.</span><span class="sxs-lookup"><span data-stu-id="89600-155">Because the grace days are now tracked at the customer level, the next collection letter won't be sent until the number of grace days has passed for the next collection letter in the sequence, even though transactions became overdue after the last collection letter was sent.</span></span> <span data-ttu-id="89600-156">Šī iespēja palīdz samazināt katram klientam nosūtāmo atgādinājuma vēstuļu skaitu.</span><span class="sxs-lookup"><span data-stu-id="89600-156">This option helps reduce the number of collection letters that you must send to each customer.</span></span>
