# <a name="invoice-capture-received-files"></a>Rēķina tveršana saņemtajiem failiem

[!include [banner](../includes/banner.md)]
[!include [preview banner](../includes/preview-banner.md)]

Rēķina tveršanas laikā lapa **Saņemtie** faili ir centrālā vieta, kur rēķina faili tiek saņemti no dažādiem avotiem.

## <a name="upload-invoice-files"></a>Augšupielādēt rēķinu failus

Parādi kreditoriem (AP) darbinieki var augšupielādēt rēķinu attēlus, **atlasot Augšupielādēt failu**, lai atvērtu **dialoglodziņu** Augšupielāde. Kad AP darbinieks atlasa failu, tiek **pieejama** poga Augšupielāde.

## <a name="include-or-obsolete-invoice-files"></a>Ietvert vai novecojušus rēķinu failus

Lietotāji var atlasīt rēķinus ar kļūdām vai brīdinājumiem un pēc tam iekļaut vai novecojušus rēķinus, darbību rūtī atlasot atbilstošo pogu. Rēķins, kas ir atzīmēts kā novecojis, netiks rādīts skatā **Saņemtie faili (Novecojis**).

## <a name="view-captured-invoices"></a>Skatīt tvertos rēķinus

Kad saņemtais fails ir veiksmīgi atpazīts, AI modelis atgrieza saglabāto rēķina informāciju un ievada to Microsoft Dataverse. Pēc tam KREDITORA darbinieks var pārbaudīt rēķina informāciju, atlasot Skatīt **saglabāto rēķinu**. Lietotāji var lejupielādēt sākotnējā rēķina failu pēc vajadzības.
