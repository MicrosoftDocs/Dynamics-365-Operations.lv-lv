# <a name="submit-invoices-to-the-workflow-system-and-match-product-receipt-lines"></a><span data-ttu-id="246e7-103">Rēķinu iesniegšana darbplūsmu sistēmā un produktu ieejas plūsmu atbilstības nodrošināšana</span><span class="sxs-lookup"><span data-stu-id="246e7-103">Submit invoices to the workflow system and match product receipt lines</span></span>

[!include [banner](../includes/banner.md)]

<span data-ttu-id="246e7-104">Šajā tēmā ir paskaidrota kreditoru rēķinu iesniegšana darbplūsmas sistēmā un automātiska grāmatoto preču ieejas plūsmas rindu saskaņošana ar kreditora rēķiniem.</span><span class="sxs-lookup"><span data-stu-id="246e7-104">This topic explains the process of submitting vendor invoices to the workflow system and automatically matching posted product receipt lines to vendor invoices.</span></span>

## <a name="submitting-imported-vendor-invoices-to-the-workflow-system-and-matching-posted-product-receipt-lines-to-pending-vendor-invoice-lines"></a><span data-ttu-id="246e7-105">Importēto kreditoru rēķinu iesniegšana darbplūsmas sistēmā un grāmatoto preču ieejas plūsmas rindu saskaņošana ar gaidošām kreditora rēķina rindām</span><span class="sxs-lookup"><span data-stu-id="246e7-105">Submitting imported vendor invoices to the workflow system and matching posted product receipt lines to pending vendor invoice lines</span></span>

<span data-ttu-id="246e7-106">Kā daļu no bezkontakta kreditoru rēķinu izrakstīšanas procesa, varat likt sistēmai automātiski iesniegt importētu rēķinu darbplūsmas sistēmā.</span><span class="sxs-lookup"><span data-stu-id="246e7-106">As part of a touchless Accounts payable invoicing process, you can have the system automatically submit an imported invoice to the workflow system.</span></span> <span data-ttu-id="246e7-107">Varat konfigurēt importēto rēķinu iesniegšanas procesu darbplūsmas sistēmā cilnē **Kreditoru rēķinu izrakstīšanas automatizācija** lapā **Parādu kreditoriem parametri** (**Kreditori \> Iestatījumi \> Parādu kreditoriem parametri**).</span><span class="sxs-lookup"><span data-stu-id="246e7-107">You can configure the process of submitting imported invoices to the workflow system on the **Vendor invoice automation** tab of the **Accounts payable parameters** page (**Accounts payable \> Setup \> Accounts payable parameters**).</span></span> <span data-ttu-id="246e7-108">Darbplūsmā iesniegšanas process darbosies fonā, izmantojot jūsu norādīto biežumu (katru stundu vai dienu).</span><span class="sxs-lookup"><span data-stu-id="246e7-108">The submit-to-workflow process will run in the background, at a frequency that you specify (either hourly or daily).</span></span>

<span data-ttu-id="246e7-109">Automātiski iesniedzot rēķinus darbplūsmas sistēmā, jāsāk ar importētu rēķinu.</span><span class="sxs-lookup"><span data-stu-id="246e7-109">When you automatically submit invoices to the workflow system, you must begin with an imported invoice.</span></span> <span data-ttu-id="246e7-110">Lai nodrošinātu, ka rēķinu var pilnībā apstrādāt bez manuālas iejaukšanās, jums darbplūsmas konfigurācijā ir jāiekļauj automatizēts grāmatošanas uzdevums.</span><span class="sxs-lookup"><span data-stu-id="246e7-110">To ensure that the invoice can be processed from start to finish without manual intervention, you must include an automated posting task in the workflow configuration.</span></span> <span data-ttu-id="246e7-111">Rēķinus, kas saistīti ar pirkšanas pasūtījumiem (PP), un rēķinus, kas ietver ar PP nesaistītu sagādes kategoriju, un neuzkrātas rindas var automātiski iesniegt darbplūsmas sistēmā.</span><span class="sxs-lookup"><span data-stu-id="246e7-111">Invoices that are related to purchase orders (POs), and invoices that contain a non-PO procurement category and non-stocked lines, can automatically be submitted to the workflow system.</span></span> <span data-ttu-id="246e7-112">Manuāli ievadītie rēķini ir manuāli jāiesniedz darbplūsmas sistēmā.</span><span class="sxs-lookup"><span data-stu-id="246e7-112">Invoices that are manually entered must be manually submitted to the workflow system.</span></span>

<span data-ttu-id="246e7-113">Darbplūsmas vērtība **Iesniedza** ir fona uzdevumā **Kreditoru rēķinu iesniegšana darbplūsmā** ievadītais lietotāja ID lapā **Procesu automatizācija**.</span><span class="sxs-lookup"><span data-stu-id="246e7-113">The **Submitted by** value in the workflow is the user ID that was entered for the **Submit vendor invoices to workflow** background task on the **Process automation** page.</span></span> <span data-ttu-id="246e7-114">Lietotājs, kam ir šis lietotāja ID, pēc nepieciešamības varēs atsaukt rēķinu no darbplūsmas sistēmas.</span><span class="sxs-lookup"><span data-stu-id="246e7-114">The user who has that user ID will be able to recall the invoice from the workflow system as required.</span></span>

## <a name="matching-posted-product-receipts-to-invoice-lines-that-have-a-three-way-matching-policy"></a><span data-ttu-id="246e7-115">Grāmatoto preču ieejas plūsmu saskaņošana ar rēķina rindām, kurām ir trīsvirzienu atbilstības ierobežojumi</span><span class="sxs-lookup"><span data-stu-id="246e7-115">Matching posted product receipts to invoice lines that have a three-way matching policy</span></span>

<span data-ttu-id="246e7-116">Kā daļu no bezkontakta kreditoru rēķinu izrakstīšanas procesa, sistēma var automātiski saskaņot grāmatotās produktu ieejas plūsmas ar rēķina rindām.</span><span class="sxs-lookup"><span data-stu-id="246e7-116">As part of a touchless Accounts payable invoicing process, the system can automatically match posted product receipts to invoice lines.</span></span> <span data-ttu-id="246e7-117">Šim uzdevumam ir jādefinē trīsvirzienu atbilstības ierobežojumi.</span><span class="sxs-lookup"><span data-stu-id="246e7-117">A three-way matching policy must be defined for this task.</span></span> <span data-ttu-id="246e7-118">Šis līdzeklis ir pieejams, ja līdzeklis **Kreditoru rēķinu izrakstīšanas automatizācija** ir iespējots lapā **Līdzekļu pārvaldība**.</span><span class="sxs-lookup"><span data-stu-id="246e7-118">This feature is available if the **Vendor invoice automation** feature has been enabled on the **Feature management** page.</span></span>

<span data-ttu-id="246e7-119">Atbilstības process tiks veikts, līdz saskaņotais preču ieejas plūsmas daudzums būs vienāds ar rēķina daudzumu.</span><span class="sxs-lookup"><span data-stu-id="246e7-119">The matching process will run until the matched product receipt quantity equals the invoice quantity.</span></span> <span data-ttu-id="246e7-120">Tomēr, ja vienai rēķina rindai ir vairākas produktu ieejas plūsmas, process ir jāpalaiž vairākas reizes, lai sasniegtu pilnu daudzuma atbilstību.</span><span class="sxs-lookup"><span data-stu-id="246e7-120">However, if there are multiple product receipts for a single invoice line, you’ll need to run the process multiple times to achieve the full quantity match.</span></span> <span data-ttu-id="246e7-121">Var norādīt maksimālo reižu skaitu, kuru laikā sistēmai jāmēģina saskaņot preču ieejas plūsmu ar rēķina rindu, pirms tiek secināts, ka process nav izdevies.</span><span class="sxs-lookup"><span data-stu-id="246e7-121">You can specify the maximum number of times that the system should try to match product receipts to an invoice line before it concludes that the process failed.</span></span> <span data-ttu-id="246e7-122">Process darbosies fonā vai nu katru stundu, vai katru dienu.</span><span class="sxs-lookup"><span data-stu-id="246e7-122">The process will run in the background, either hourly or daily.</span></span> 

<span data-ttu-id="246e7-123">Automatizēto saskaņošanas procesu var palaist kā daļu no rēķinu iesniegšanas procesa darbplūsmas sistēmā.</span><span class="sxs-lookup"><span data-stu-id="246e7-123">You can run the automated matching process as part of the process for submitting invoices to the workflow system.</span></span> <span data-ttu-id="246e7-124">Vai arī varat to palaist kā savrupu procesu.</span><span class="sxs-lookup"><span data-stu-id="246e7-124">Alternatively, you can run it as a standalone process.</span></span> <span data-ttu-id="246e7-125">Iestatījumi preču ieejas plūsmu un rēķina rindu saskaņošanas procesam tiek konfigurēti cilnē **Kreditoru rēķinu izrakstīšanas automatizācija** lapā **Parādu kreditoriem parametri** (**Kreditori \> Iestatījumi \> Parādu kreditoriem parametri**).</span><span class="sxs-lookup"><span data-stu-id="246e7-125">The settings for the match-product-receipts-to-invoice-lines process are configured on the **Vendor invoice automation** tab of the **Accounts payable parameters** page (**Accounts payable \> Setup \> Accounts payable parameters**).</span></span>

<span data-ttu-id="246e7-126">Rēķina rindas, kurām ir trīsvirzienu atbilstības ierobežojumi, kur saskaņotais ieejas plūsmas daudzums ir mazāks par daudzumu rēķinā, tiks iekļauts automatizētajā saskaņošanas procesā ar preču ieejas plūsmu.</span><span class="sxs-lookup"><span data-stu-id="246e7-126">Invoice lines that have a three-way matching policy, where the matched receipt quantity is less than the invoice quantity, will be included in the automated match-to-product-receipt process.</span></span>

<span data-ttu-id="246e7-127">Lai skatītu statusu **Pēdējā atbilstība** rēķiniem, kas nav iekļauti automatizētajā darbplūsmā iesniegšanas procesā, atveriet rēķinu no lapas **Kreditoru rēķini**.</span><span class="sxs-lookup"><span data-stu-id="246e7-127">To view the **Last match** status for invoices that aren't part of the automated submit-to-workflow process, open the invoice from the **Vendor invoices** page.</span></span> <span data-ttu-id="246e7-128">Skatot rēķinu, atbilstošā validācijas informācija tiek atjaunināta.</span><span class="sxs-lookup"><span data-stu-id="246e7-128">When you view the invoice, the matching validation information is updated.</span></span> <span data-ttu-id="246e7-129">Statusu **Pēdējā atbilstība** var atjaunināt automātiski, izmantojot fona uzdevumu **Pārbaudīt rēķina atbilstību**.</span><span class="sxs-lookup"><span data-stu-id="246e7-129">The **Last match** status can be updated automatically using the **Validate invoice matching** background task.</span></span> <span data-ttu-id="246e7-130">Varat konfigurēt statusa **Pēdējā atbilstība** sautomātiskās atjaunināšanas procesu lapas **Procesu automatizācija** cilnē **Fona procesi** (**Sistēmas adminstrēšana\> Iestatīšana\> Procesa automatizācija**).</span><span class="sxs-lookup"><span data-stu-id="246e7-130">You can configure the process of automatically updating the **Last match** status on the **Background processes** tab of the **Process automations** page (**System adminstration\> Setup\> Process automations**).</span></span>

<span data-ttu-id="246e7-131">Rēķina rinda tiks izslēgta no automātiskās apstrādes, ja izpildīts kāds no šiem nosacījumiem:</span><span class="sxs-lookup"><span data-stu-id="246e7-131">An invoice line will be excluded from the automated processing if any of the following conditions are met:</span></span>

- <span data-ttu-id="246e7-132">Rēķina rindas vērtība **Automatizētās preču ieejas plūsmas atbilstības statuss** ir **Nesekmīgs**.</span><span class="sxs-lookup"><span data-stu-id="246e7-132">The **Automated receipt match status** value of the invoice line is **Failed**.</span></span>
- <span data-ttu-id="246e7-133">Rēķins tiek izmantots.</span><span class="sxs-lookup"><span data-stu-id="246e7-133">The invoice is being used.</span></span>
- <span data-ttu-id="246e7-134">Rēķins ir darbplūsmas sistēmā.</span><span class="sxs-lookup"><span data-stu-id="246e7-134">The invoice is in the workflow system.</span></span>
